# Data dictionary

This repository contains two scoring data files. One is the **reported summary**
transcribed from the manuscript; the other is a **derived, reconstructed**
criterion-level expansion. Read the provenance notes carefully — they are not
equivalent kinds of record.

---

## `../scoring_data.csv` — reported summary data

Two rows per FCI item (one per condition), transcribed from the manuscript's
**Table III (Raw Data)**. The numeric values reproduce Table III exactly.

| Column | Type | Meaning |
|--------|------|---------|
| `fci_question` | integer | FCI item number (5, 16, 17, 22, 23) |
| `condition` | string | `Constrained` or `Unconstrained` |
| `criteria_met` | integer | Number of applicable rubric criteria satisfied |
| `applicable_criteria` | integer | Number of the 17 rubric criteria applicable to this item (14–17) |
| `accuracy_percent` | string | `criteria_met / applicable_criteria` × 100, two decimals |
| `failed_criteria_count` | integer | Number of applicable criteria not met |
| `failed_criteria` | string | `; `-separated rubric criterion names that failed; empty when none failed |

`failed_criteria` uses the exact canonical rubric criterion names (see below) so
the file is machine-checkable against the derived criterion-level file.

---

## `criterion_level_scoring.csv` — derived criterion-level scoring

> **Provenance: this file is DERIVED, not an original scoring sheet.** It was
> deterministically reconstructed from `scoring_data.csv` (the reported summary)
> and the published 17-criterion rubric (manuscript Table II) plus the reported
> per-item failed-criterion lists (manuscript Table III). It is **not** a
> contemporaneous raw scoring sheet captured while the study was run. It is
> regenerated by `scripts/build_criterion_level_scoring.py` and verified by
> `scripts/validate_research_data.py`.

One row per (FCI question × condition × rubric criterion): 5 × 2 × 17 = **170 rows**.

| Column | Type | Meaning |
|--------|------|---------|
| `fci_question` | integer | FCI item number (5, 16, 17, 22, 23) |
| `condition` | string | `Constrained` or `Unconstrained` |
| `criterion_number` | integer | Rubric criterion index, 1–17 (see list below) |
| `criterion` | string | Rubric criterion text (manuscript Table II wording) |
| `applicable` | boolean | `true` / `false` — whether the criterion applies to this item's setup |
| `result` | string | `Pass`, `Fail`, or `Not applicable` |
| `provenance` | string | Always: `Reconstructed from scoring_data.csv and the published rubric` |
| `notes` | string | Why a criterion is not applicable, or the basis of the pass/fail |

`result` is `Not applicable` if and only if `applicable` is `false`.

### The 17 rubric criteria

1. Correct setup (projectile-motion or free-fall)
2. Modifiable mass
3. Modifiable initial height
4. Modifiable initial velocity
5. Modifiable launch angle — *projectile-motion only*
6. Enable/disable view of force_gravity vector and components
7. Enable/disable view of force_air_drag vector and components — *air-resistance setups only*
8. Enable/disable view of velocity and components
9. Enable/disable view of acceleration vector
10. Enable/disable view of motion tracking
11. Speed options
12. Time scrubbing
13. View of key frames in time
14. Four or more key motion graphs
15. Net acceleration is always -g
16. Horizontal velocity constant — *projectile-motion only*
17. Graphs consistent with expected qualitative physics relationships

### Applicability rules

- **Q5** (free fall, no air resistance): all criteria except **5, 7, and 16** → **14 applicable**.
- **Q17** (free fall): all criteria except **5 and 16** → **15 applicable**.
- **Q16, Q22, Q23** (projectile motion): all **17** apply.

Criteria 5 and 16 are projectile-motion-only; criterion 7 applies only to
setups that include air resistance.

### How to regenerate and validate

```bash
python scripts/build_criterion_level_scoring.py   # writes data/criterion_level_scoring.csv
python scripts/validate_research_data.py          # exits nonzero on any disagreement
```

The validator recomputes `criteria_met`, `applicable_criteria`,
`accuracy_percent`, `failed_criteria_count`, and the failed-criterion set from
the criterion-level file and confirms exact agreement with both
`scoring_data.csv` and an independent hard-coded copy of the Table III values.
